# [ORG_NAME] Org Coordination Repo

## What This Repo Is

This is the org-level coordination hub for the [ORG_NAME] OpenJunto Codex configuration. It holds shared reference material, an org-level backlog, cross-project context maps, and slash commands that span across the individual project repos this org manages. It is **not** a per-developer Codex installation — those live in `~/.codex/` and are managed separately by each developer.

## How It Relates to Per-Project `.codex/` Directories

OpenJunto uses a three-tier inheritance model:

```
Tier 1: Global OpenJunto    (~/.codex/ — installed per developer)
  ↓ always loaded
Tier 2: Org                 (this repo — cross-project coordination hub)
  ↓ loaded when operating from this repo or referencing org state
Tier 3: Per-project         (.codex/ in each member project repo)
  ↓ always loaded when in a project directory
```

This org repo is a **source of shared reference**, not a config directory that Codex reads automatically. Developers load org reference files explicitly (via Read tool or by copying to `~/.codex/reference/`). The org repo does NOT inject a protocol file into the context chain — it provides shared state and reference only.

## How to Use It

**Finding the backlog**: Open `BACKLOG.md` in this repo for cross-project and org-level work items. Per-project backlogs live in each project's `.codex/BACKLOG.md`.

**Running org-level commands**: Point Codex at this directory and run a slash command. Example — to run the org health check:

```
# In your terminal, navigate to this repo:
cd [ORG_REPO_PATH]

# Then in your Codex session:
/health-check
```

**Referencing org context from a project**: When working in a member project and you need org-level facts, explicitly read the relevant file:

```
# In your Codex session (from a member project):
Read [ORG_REPO_PATH]/reference/validated-facts-org.md
```

**Adding a new project**: See `reference/org-coordination-guide.md` § Adding a New Project.

**Updating shared facts**: When a validated fact is confirmed across 2+ member projects, add it to `reference/validated-facts-org.md` following the format in that file.
