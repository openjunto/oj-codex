# /health-check (Org Scope)

Verify that the org coordination repo and all coordinated projects are in coherent state. Read-only — no modifications. Observations only, no auto-fix.

## Steps

### 1. Org Repo State

Check the following and report PASS / WARN / FAIL for each:

- **`llms.txt` present and non-empty**: Read `llms.txt`. If missing → FAIL. If present but contains only placeholder entries with no real project paths → WARN with message "llms.txt has no active project entries."
- **`BACKLOG.md` present**: Read `BACKLOG.md`. If missing → FAIL.
- **`state/` gitignored**: Read `.gitignore`. Confirm `state/*` and `!state/.gitkeep` are both present. If either is missing → FAIL. If both present → PASS.

### 2. Project Inventory

Read `llms.txt`. For each project entry under `## Projects`:

- If the entry is a local path: check whether the path exists on the filesystem. If it resolves → PASS. If it does not resolve → WARN with "Path not found: [path] — update llms.txt or verify the repo location."
- If the entry is a URL: note it as "remote — path check skipped" (PASS with note).
- If no project entries exist (only placeholders): WARN "No active projects listed in llms.txt."

### 3. Stale State Detection

Check `state/` directory:

- Run: does `state/session.md` exist AND is it tracked by git (not gitignored)? If it is git-tracked → FAIL "state/session.md is tracked by git — add to .gitignore or remove from tracking."
- If `state/session.md` is present but gitignored → PASS (expected behavior).
- If `state/` is empty except for `.gitkeep` → PASS.

### 4. Org Backlog Coherence

Read `BACKLOG.md`:

- Count open items (items under `## Active` with `**Status**: Open` or `**Status**: In Progress`).
- Scan for items marked `**Status**: In Progress` that have no owner field — WARN for each: "Item [BACK-NNN] is In Progress but has no owner."
- Scan for items that reference other BACK-NNN items in their description that are marked Completed — WARN: "Item [BACK-NNN] may be unblocked — its blocker [BACK-MMM] is completed."
- Report total open item count regardless.

### 5. Reference Freshness Signal

Check `reference/validated-facts-org.md`:

- Report the file's last-modified date if detectable (via filesystem metadata or git log).
- If last modified more than 30 days ago → WARN "validated-facts-org.md last updated [N] days ago — consider reviewing."
- If last modified within 30 days → PASS.
- If the file is empty (contains only the scaffold template with no real entries) → PASS with note "No org-wide facts recorded yet."

## Output Format

Present a structured report with one line per check:

```
[PASS] llms.txt present (N projects listed)
[PASS] BACKLOG.md present (N open items)
[PASS] state/ gitignored
[WARN] validated-facts-org.md last updated 47 days ago — consider reviewing
[FAIL] state/session.md is tracked by git — add to .gitignore
```

Conclude with a summary line:

```
Health check complete: N pass, N warn, N fail.
```

If any FAIL results: "Action required before this repo is in coordination-ready state."
If only WARN results: "Advisory items noted — no blocking issues."
If all PASS: "Org coordination repo is in coherent state."

## Constraints

- Read-only. Do not modify any files.
- Do not auto-fix any detected issue — report and stop.
- Do not load per-project `.codex/` directories unless explicitly checking a path from `llms.txt`.
